当所有工具都在争取用户注意力时，胜出者是那些最"懂分寸"的。

### KAIROS 的设计哲学

```python
class RespectfulAgent:
    """
    KAIROS 的设计原则：有用但不侵入
    """
    
    async def decide(self, context) -> Action:
        if not self.has_high_value_action(context):
            return Sleep(reason="无事值得打扰用户")
        
        if not self.within_budget(context):
            return Defer(reason="会阻塞用户，推迟")
        
        if user_is_focusing():
            return NotifyAsync(reason="用户专注中，异步通知")
        
        return Act(action=self.best_action())
```

    },
    {
      "timestamp": 1743601000,
      "type": "inference",
      "content": "用户似乎在使用 JWT，可能正在做认证系统"
    }
  ]
}
```

### 隐私问题清单

| 问题 | 严重性 | 可能的解决方案 |
|------|--------|----------------|
| 日志包含代码内容 | 🔴 高 | 仅存储元数据，不存代码 |
| 推断用户意图 | 🔴 高 | 用户控制推断深度 |
| 跨项目关联 | 🟡 中 | 项目间隔离 |
| 长期存储 | 🟡 中 | 自动过期、本地加密 |
| 谁可以访问 | 🔴 高 | 零知识架构、本地优先 |

### 零知识 KAIROS 构想

```
┌─────────────────────────────────────────────────────────────┐
│                   零知识 KAIROS 架构                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   本地设备                       云端（如果有）             │
│   ┌─────────────────┐           ┌─────────────────┐        │
│   │ 观察日志（加密） │           │ 加密存储        │        │
│   │ 索引（本地）     │◄─────────→│ 无法解密        │        │

│   └─────────────────┘           └─────────────────┘        │
│                                                             │
│   原则：                                                     │
│   1. 观察日志永远本地存储                                    │
│   2. 云端备份为加密状态，厂商无法访问                        │
│   3. 用户完全控制数据生命周期                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
